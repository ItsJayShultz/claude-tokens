01 How Claude's Limits Work

There are two separate limits running at the same time. Most people only know about one.

5-Hour Limit

A rolling window that resets every 5 hours. This is the one you hit most often — heavy sessions drain it fast.

7-Day Limit

A weekly cap that resets every 7 days. Harder to hit, but uploading large files repeatedly will get you there.

The hidden problem: Every message in a conversation gets re-read by Claude every single time you send a new one. The longer your conversation, the more tokens each reply costs — even if you only asked one short question.

02 What Costs the Most Tokens

Not all inputs are equal. Some formats burn through your limit 2x faster than others.

Token Cost by Input Type
Raw PDF Full text extraction, every word counted ~3,000 tokens
Screenshot of doc Image + OCR processing ~1,500 tokens
Markdown file Plain text, no overhead ~500 tokens
Long conversation Full history re-read every reply Compounds fast
Google Doc (pasted) Plain text, no formatting noise Efficient

03 Use the Right File Format

Switching from PDF to markdown is the single biggest token saving most people can make.

The Simple Rule
  • Never upload a raw PDF if you can avoid it — it costs ~3,000 tokens per document
  • Take a screenshot of the document instead — cuts the cost roughly in half (~1,500 tokens)
  • Better still: open the doc in Google Docs → File → Download → Markdown (.md) — then drag that into Claude
  • Or just copy-paste the text directly — plain text is the cheapest input of all
How to Export as Markdown from Google Docs
  • 1Open your document in Google Docs
  • 2Click File → Download → Markdown (.md)
  • 3Drag the downloaded .md file into Claude
  • 4Same content, fraction of the token cost

04 Track Your Token Usage

You can't manage what you can't see. Here's how to monitor your usage in real time.

Built-in Claude Usage Bar

At the bottom of the Claude chat window you can see two bars:

  • Session: your 5-hour rolling usage (e.g. "48% · resets in 2h 57m")
  • Weekly: your 7-day usage (e.g. "13% · resets in 6d 0h")

Check this before starting a heavy session so you know how much runway you have.

Claude Counter Chrome Extension

Install Claude Counter from the Chrome Web Store. It shows you exactly how many tokens each message costs, in real time.

  • See token count per message as you type
  • Track cumulative session usage
  • Instantly see which inputs are burning your limit

05 Caveman Mode

A custom instruction that cuts token usage by ~75% by having Claude respond in ultra-compressed language — while keeping full technical accuracy.

What is Caveman Mode?

A Claude customisation (available on GitHub: JuliusBrussee/caveman) that makes Claude drop filler words, articles, and padding. Instead of "Sure! I'd be happy to help you with that. The issue you're experiencing is..." — you get "Fix: [solution]."

Intensity Levels
LevelWhat it does
liteNo filler or hedging. Keeps articles and full sentences. Professional tone.
fullDrops articles, fragments OK, short synonyms. Classic caveman.
ultraAbbreviates everything (DB, auth, config, req, res). Strip to bare minimum.
wenyan-liteSemi-classical style. Drop filler but keep grammar structure.
How to activate it
  • 1Go to Claude → Customizations (or Custom Instructions)
  • 2Name it caveman-token-reducer
  • 3Paste the instructions from the GitHub repo: github.com/JuliusBrussee/caveman
  • 4Trigger it by typing "use caveman", "less tokens", or "be brief" in any chat
Note: Caveman mode automatically turns off for security warnings, irreversible actions, and multi-step operations where clarity matters. It's smart about when to stay compressed.

06 Shortcut Codes

Prefix these words before your prompt to change how Claude responds — shorter, sharper, more useful output every time.

11 Shortcut Codes That Save Tokens
BLUNT: No hedging, no caveats, no disclaimers. Just the answer. "BLUNT: Should I use React or Next.js?"
BRUTAL: Brutal honesty, no compliment sandwich. Tell me exactly what's wrong. "BRUTAL: Review my résumé and tell me why I'm not getting interviews."
ASSUME: List all hidden assumptions you're making before answering. "ASSUME: How long will it take me to learn machine learning?"
COUNTER: Generate the strongest counterargument to my point or plan. "COUNTER: I think building in public is always good for a startup."
CONTRAST: Force an explicit side-by-side comparison before concluding. "CONTRAST: MongoDB vs PostgreSQL for a social media app."
SCOPE: Define and constrain the exact problem space before solving it. "SCOPE: How do I improve my app's performance?"
ZOOM: Alternate between tactical detail and strategic altitude. "ZOOM: How should I approach learning system design?"
POST: Post-mortem mode — analyse what went wrong and why, no sugarcoating. "POST: My side project got no users after 3 months. Here's what I did."
2ND_FX: Map the cascade effects of this decision two levels deep. "2ND_FX: What happens if I drop out of college to start a company?"
LADDER: Answer at three levels — beginner, intermediate, expert — in that order. "LADDER: Explain how transformers work in AI."
STEELMAN: Give me the strongest possible version of the opposing argument. "STEELMAN: The case against using AI tools as a student."

07 Quick Wins — Do These Today

Simple habits that save tokens every single session without changing how you work.

7 Habits That Cut Your Usage in Half
  • 1Start a new conversation for each new topic — don't let one chat grow for days
  • 2Use markdown files instead of PDFs whenever possible
  • 3Paste text directly instead of uploading documents for short content
  • 4Install Claude Counter to see what's actually costing you
  • 5Use BLUNT: or BRUTAL: prefixes to get shorter, direct answers
  • 6Set up Caveman Mode for any technical or repetitive work
  • 7Check your usage bars before starting a heavy session
